# Content Design Principles

Use this skill as a final writing pass before producing user-facing text. Preserve required technical detail, file paths, commands, citations, and caveats, but make the communication easier to act on.

## Principles

- Start from the user need. Write what the reader needs to know to do or decide something, not what you want to say.
- Front-load everything. Put the most important point first in the whole answer, each section, each paragraph, and each sentence. Use the inverted pyramid: conclusion, then detail, then background.
- Use one idea per sentence and one topic per paragraph. Split sentences that carry more than one idea.
- Be specific and concrete. Give the number, name, date, command, path, URL, error string, or result when it matters.
- Cut vague abstractions. Avoid phrases like "a range of", "going forward", "in terms of", "various", "some", "things", and "stuff" when a concrete noun is available.
- Cut anything that does not add meaning. Shorter is clearer. Remove duplication.

## Output Check

Before sending user-facing text:

1. Put the answer or decision first.
2. Remove setup sentences that only describe your intent.
3. Replace vague wording with concrete evidence.
4. Split overloaded sentences.
5. Delete repeated context.
6. Keep the tone direct, useful, and calm.
